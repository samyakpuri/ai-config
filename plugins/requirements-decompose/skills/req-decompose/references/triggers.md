# Trigger Phrases

Full list of phrases and scenarios that should load this skill, grouped by use case.

## Requirements extraction / decomposition

- "decompose requirements"
- "break down this spec"
- "generate requirements from this doc"
- "extract firmware requirements"
- "write component requirements"
- "turn this into structured requirements"
- "requirements traceability"
- "parse this PRD"
- "I have a product spec" / "I have a PRD" / "here's the product brief"
- "spec this feature"
- "use this Polarion export"
- "populate component templates"
- any request to convert a raw document into layered SYS/FW/SW/UI/API/COMP requirements with traceability

## Baseline comparison / delta analysis

- "compare requirements against baseline"
- "what changed since last requirements doc"
- "diff requirements"
- "requirements delta"
- "identify outdated requirements"
- "what's new in this spec"
- "update system requirements from product brief"
- "what is missing from our system requirements"
- any request to compare a new or updated requirements document against an existing set of system requirements

## Input forms that imply this skill

- providing a `.pdf` (especially Polarion exports) alongside a decomposition request
- providing a `.docx` or `.md` product brief with a request to structure or extract requirements
- mentioning Polarion, DOORS, or structured requirement IDs (REQ-NNN, SWR-NNN) in context of requirements work
- mentioning two documents — a product doc and an existing SYS requirements doc — with a request to reconcile them
